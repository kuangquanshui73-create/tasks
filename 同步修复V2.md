# 同步功能修复 V2 (2026-03-20)

## 修复内容

### ✅ 问题 1: 自动同步只上传不下载
**修复位置:** `app.js` 第1617行

**修改前:**
```javascript
this.autoSyncInterval = setInterval(() => {
    this.syncToCloud();  // 只上传!
}, 5 * 60 * 1000);
```

**修改后:**
```javascript
this.autoSyncInterval = setInterval(async () => {
    try {
        // 先从云端获取最新数据
        await this.syncFromCloud();
        // 再上传本地数据
        await this.syncToCloud();
    } catch (error) {
        console.error('自动同步失败:', error);
    }
}, 5 * 60 * 1000);
```

**效果:** 每5分钟自动执行双向同步,确保数据在所有设备间保持一致。

### ✅ 问题 2: 登录后不立即同步
**修复位置:** `app.js` 第1218行

**修改前:**
```javascript
if (this.token) {
    this.startAutoSync();  // 只启动定时器
}
```

**修改后:**
```javascript
if (this.token) {
    // 先从云端同步数据,再启动自动上传
    this.syncFromCloud().then(() => {
        this.startAutoSync();
    }).catch(() => {
        // 如果首次同步失败(云端没有数据),仍然启动自动同步
        this.startAutoSync();
    });
}
```

**效果:** 用户打开应用后,会立即从云端获取最新数据,无需等待5分钟或手动点击同步。

### ✅ 问题 3: 数据冲突处理
**修复位置:** `app.js` 第1567行 `mergeCloudData()` 方法

**修改前:**
```javascript
// 只是添加不存在的任务
const newTasks = cloudData.tasks.filter(t => !existingIds.includes(t.id));
if (newTasks.length > 0) {
    this.taskManager.tasks = [...this.taskManager.tasks, ...newTasks];
}
```

**修改后:**
```javascript
// 根据最后修改时间决定使用哪个版本
cloudData.tasks.forEach(cloudTask => {
    const localTask = existingMap.get(cloudTask.id);

    if (!localTask) {
        // 新任务,直接添加
        this.taskManager.tasks.push(cloudTask);
    } else {
        // 任务已存在,需要决定使用哪个版本
        const cloudModified = new Date(cloudTask.lastModified || cloudTask.completedAt || cloudTask.createdAt || 0);
        const localModified = new Date(localTask.lastModified || localTask.completedAt || localTask.createdAt || 0);

        if (cloudModified > localModified) {
            // 云端版本更新,使用云端版本
            const index = this.taskManager.tasks.findIndex(t => t.id === cloudTask.id);
            if (index !== -1) {
                this.taskManager.tasks[index] = cloudTask;
            }
        }
        // 如果本地版本更新,保留本地版本
    }
});
```

**效果:** 当两台设备同时修改同一任务时,会自动保留最后修改的版本,避免数据丢失。

## 使用说明

### 在电脑上使用:

1. 打开 `index.html`
2. 登录 Gitee 云同步(如果还没登录)
3. 添加或修改任务
4. **应用会自动同步**:
   - 修改后2秒自动上传
   - 每5分钟自动双向同步
   - 打开应用时立即从云端拉取数据

### 在手机上使用:

1. 打开 https://kuangquanshui73-create.github.io/tasks
2. 登录 Gitee 云同步
3. **等待5-10秒**,数据会自动从云端同步下来
4. 添加或修改任务,也会自动同步

### 手动同步:

如果自动同步没有生效,可以:
1. 点击页面底部的"立即同步"按钮
2. 或者刷新页面(会触发登录后立即同步)

## 常见问题

### Q: 为什么手机上看不到电脑上的任务?

**A:** 可能的原因:
1. 没有登录 Gitee 云同步 → 登录后再试
2. 网络问题 → 检查网络连接
3. 等待时间不够 → 等待5-10秒或手动点击"立即同步"
4. 浏览器缓存 → 尝试强制刷新(Ctrl+F5)

### Q: 为什么数据不一致?

**A:** 检查方法:
1. 使用 `diagnose-sync.html` 对比本地和云端数据
2. 点击"立即同步"手动同步
3. 如果仍然不一致,可能是冲突导致,最后修改的版本会保留

### Q: 自动同步太慢,能加快吗?

**A:** 可以修改同步间隔:
在 `app.js` 第1619行,将 `5 * 60 * 1000` 改为更小的值,例如:
```javascript
// 每1分钟同步
}, 1 * 60 * 1000);

// 每30秒同步
}, 30 * 1000);
```

**注意:** 同步频率过高会增加网络请求和服务器负载。

## 技术细节

### 数据流图

```
设备 A                    Gitee 云端              设备 B
  |                        |                       |
  |-- 上传数据 ------------>|                       |
  |                        |-- 保存数据            |
  |                        |                       |
  |                        |-- 同步通知 --------->|
  |                        |                       |-- 下载并显示
  |                        |                       |
  |<------- 下载数据 -------|<------ 上传数据 ------|
  |-- 合并并显示           |<-- 保存并更新         |
```

### 同步触发时机

1. **修改数据后2秒** - 自动上传到云端
2. **每5分钟** - 完整双向同步(下载+上传)
3. **页面打开时** - 如果已登录,立即从云端下载
4. **页面关闭时** - 自动上传本地数据
5. **手动点击** - 立即执行上传

### 冲突解决策略

使用"最后写入者胜"(Last-Write-Wins)策略:
- 比较 `lastModified` 时间戳
- 保留时间戳较新的版本
- 这种策略简单且适合大多数使用场景
