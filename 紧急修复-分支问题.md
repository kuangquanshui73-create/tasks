## 技术细节

### 为什么会出现这个问题?

Gitee 仓库的默认分支是 `master`，但代码中写的是 `main`。

GitHub 在 2020 年将默认分支从 `master` 改为 `main`，但很多老的 Gitee 仓库还是使用 `master`。

当你尝试向 `main` 分支创建文件时，Gitee 返回错误：
```json
{
  "message": "只允许在分支上创建或更新文件"
}
```

### 修复方案

将代码中的 `this.branch = 'main'` 改为 `this.branch = 'master'`
